## 1. Reto de API RESTful: Sistema de Tareas (To-Do List)

Este es un reto fundamental para evaluar el dominio de los conceptos básicos de una API.

Descripción:
Crea una API RESTful para gestionar una lista de tareas. Un usuario debe poder gestionar sus propias tareas, pero no las de otros.

Requisitos Clave:

Autenticación: Usa Laravel Sanctum para autenticar a los usuarios a través de tokens de API.

Endpoints (Rutas):

POST /api/tasks (Crear una tarea. Columnas: title, description, due_date, status).

GET /api/tasks (Listar solo las tareas del usuario autenticado).

GET /api/tasks/{task} (Ver una tarea específica).

PUT /api/tasks/{task} (Actualizar una tarea).

DELETE /api/tasks/{task} (Eliminar una tarea).

Validación: Utiliza Form Request Classes para validar todos los datos de entrada.

Seguridad: Asegúrate de que un usuario solo pueda ver, actualizar o eliminar las tareas que le pertenecen (usando Policies o validación en el controlador).

Respuestas: Formatea todas las respuestas JSON usando Eloquent API Resources para mantener una estructura consistente.

¿Qué evalúa?

Dominio de rutas de API y controladores.

Conocimiento de autenticación con Sanctum.

Buenas prácticas de validación.

Uso correcto de la autorización (Policies).

Transformación de datos para respuestas de API.
